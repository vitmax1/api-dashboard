# Техническое задание: Разработка на PHP (Laravel) и JavaScript

## Этап 1: Разработка Backend-части на Laravel
**Задача:** Создать функционал для периодического сбора данных и API-эндпоинт.

1.  **База данных:** Создать миграцию для таблицы `jokes` (id, type, setup, punchline, external_id, created_at, updated_at).
2.  **Консольная команда:** 
    *   Создать Artisan-команду (например, `app:fetch-jokes`).
    *   Команда должна обращаться к API `[https://official-joke-api.appspot.com/random_joke](https://official-joke-api.appspot.com/random_joke)`.
    *   Реализовать сохранение полученных данных в БД.
    *   Настроить запуск команды через `Schedule` (каждые 5 минут).
3.  **API Route:**
    *   Создать контроллер и маршрут (GET), который возвращает все записи из таблицы `jokes` в формате JSON.

## Этап 2: JavaScript — Динамическое управление формами
**Задача:** Написать скрипт для фильтрации видимости полей на странице.
**Контекст:** Страница содержит выпадающий список (Тип) и набор инпутов.

**Требования:**
1.  Написать JS-код (сниппет или подключаемый файл).
2.  **Логика:** При изменении значения в поле "Тип", на странице должны оставаться видимыми только те поля, в атрибуте `name` которых содержится выбранное значение. Остальные должны быть скрыты.
3.  **Аналитика:** Перечислить альтернативные алгоритмы решения этой задачи (например, через Data-атрибуты или Virtual DOM) и обосновать выбранный вариант.
4.  **Библиотеки:** Допускается использование сторонних библиотек (например, jQuery или Alpine.js), но необходимо обосновать их целесообразность.

## Этап 3: Дополнительное задание — Система статистики
**Задача:** Создать автономный счетчик посещений.

**Компонент 1 (Клиентский JS):**
*   Скрипт должен собирать: IP-адрес (через сторонний сервис или сервер), город и тип устройства.
*   Отправлять данные на бэкенд через асинхронный запрос.

**Компонент 2 (Backend & Dashboard):**
*   **Хранение:** БД SQLite или MySQL.
*   **Визуализация:** 
    *   График посещений по часам (Ось X — время, Ось Y — количество уникальных посещений).
    *   Круговая диаграмма (Pie Chart) с распределением по городам.
*   **Доступ:** Страница статистики должна быть защищена авторизацией.
